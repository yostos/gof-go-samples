# GoF Design Patterns — Java vs Go

GoFの23のデザインパターンについて、Javaの実装とGo言語での対応実装を並べて比較するサンプル集です。

本リポジトリはブログ記事シリーズ「[GoFデザインパターンは現代でも有用か — Go言語から再考する](https://codedchords.dev/blog/2026/04/gof-design-patterns/)」の付属コードです。

## 趣旨

GoFのデザインパターンは、OOP（オブジェクト指向プログラミング）の設計ノウハウとして1994年に体系化されました。しかし、OOPの中核 — クラス継承、型階層、参照セマンティクス — を意図的に排除したGo言語の登場により、23のパターンすべてがそのまま必要なわけではなくなっています。

本リポジトリでは、『[Java言語で学ぶデザインパターン入門](https://amzn.to/4bW36Po)第3版』(結城浩著) のJavaサンプルコードと同じ題材を用いて、Go言語ではどのように実装するのが最適かを示します。Javaの実装をそのまま移植するのではなく、Go言語の特性を活かした実装に書き換えることで、パターンの本質と言語依存部分を浮き彫りにします。

## 実装方針

Go言語にはクラスや継承がなく、代わりにインターフェース、構造体の埋め込み（composition）、第一級関数、チャネル/ゴルーチンといった独自のメカニズムを持っています。また、変数への代入がデータそのものをコピーする値セマンティクスがデフォルトです。

これらの言語特性を踏まえ、本リポジトリでは以下の方針でGo実装を作成しています。

### 基本方針

1. **Javaソースと同等の機能をGo言語で実装する** — 各パターンが扱う題材（じゃんけん、ファイルシステム、描画など）と動作はJava版を踏襲する
2. **Go言語が言語レベルで解決しているパターンや、現代では推奨されないパターンについては、Go言語のイディオムや現代の実装手法を採用する** — ただし、実装する機能（入出力、動作）はJava版と同等にする
3. **上記に該当しないパターンは、パターン構造をGoで忠実に再現する** — Goのインターフェースと構造体の埋め込みを用いて、Javaのクラス階層に相当する設計を表現する
4. **いずれの場合も、Javaソースの機能要素に欠落がないようにする**

### パターンごとの分類

上記方針に基づき、23のパターンを以下のように分類しています。

#### Go言語のイディオムで実装するパターン（12パターン）

Go言語が持つメカニズムにより、Javaのようなパターン構造を組む必要がないものです。Go言語のイディオムを活用しつつ、Java版と同じ機能を実現します。

| Go言語のメカニズム | パターン | 採用するイディオム |
|---|---|---|
| クラス継承がない → 合成と暗黙的インターフェースで代替 | Template Method, Factory Method, Abstract Factory, Bridge | インターフェース + 関数 |
| 値セマンティクスがデフォルト | Prototype, Memento | 構造体コピー |
| 第一級関数・チャネル・ゴルーチンが組み込み | Strategy, Command, Visitor, Observer | 関数値、クロージャ、チャネル |
| イテレーションが言語機能 | Iterator | Go 1.23 `iter.Seq`（range-over-function） |
| グローバル状態は現代の設計で非推奨 | Singleton | 依存性の注入（DI） |

#### パターン構造を忠実に再現するパターン（11パターン）

OOPに依存しない普遍的な設計原則であり、Go言語でもそのまま有用なパターンです。Goのインターフェースと構造体の埋め込みを用いて、Javaのパターン構造を忠実に再現します。

| 設計原則 | パターン |
|---|---|
| 小さな部品を合成して大きな振る舞いを作る | Composite, Decorator, Chain of Responsibility, Builder |
| コンポーネント間の境界を制御する | Adapter, Facade, Proxy, Mediator |
| 状態に応じて振る舞いを切り替える | State |
| 特定領域の問題を解決する | Flyweight, Interpreter |

## リポジトリ構成

```
src/
  <PatternName>/
    Java/          # Javaサンプルコード（結城浩著『デザインパターン入門』より）
    Go/            # Go言語での対応実装
      main.go
```
